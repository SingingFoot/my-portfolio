              <span style={{ fontSize: '9px', display: 'block', fontWeight: '400', marginTop: '1px', color: c.muted }}>{tab.desc}</span>
            </button>
          ))}
        </div>

        {scaleTab === 'instances' && (
          <div>
            <CodeBlock tokens={namedInstancesLines} />
            {explanationCard('Fine for two — awkward for a fleet', 'Pump1 and Pump2 read naturally because there are only two of them. Stretch this pattern to ten pumps and the VAR block alone needs ten declaration lines (Pump1 … Pump10), the start/stop logic must be written or copy-pasted ten times with a different instance name each time, and adding an eleventh pump means touching the declarations and duplicating the logic block again. The instance name carries the device identity, so the device identity is baked into the source code itself.', c.red)}
          </div>
        )}

        {scaleTab === 'array' && (
          <div>
            <CodeBlock tokens={arrayOfStructLines} />
            {explanationCard('ARRAY[1..10] OF Pump — one line, ten complete pumps', 'This single VAR line creates ten independent Pump instances at once — 10 × 11 = 110 typed fields — the same way ARRAY[1..5] OF BOOL created five independent BOOLs on the Arrays page. Each element Pumps[n] is not a plain value but a full Pump structure, with all eleven of its fields present and addressable. Raising the bound to [1..50] would create a 50-pump fleet without adding a single new name to the program.', c.blue)}
            {explanationCard('Pumps[4].Manuel_Start — index, then dot', 'Reading the expression left to right: Pumps[4] uses the array index to step to element 4 — "the fourth pump" — and .Manuel_Start then uses the dot to step into that pump’s structure and reach its start signal. The two features stack cleanly: indexing answers "which device", the dot answers "which signal on that device", and Pumps[4].Coil := 1; assigns to that one pump’s coil, leaving the other nine completely untouched.', c.kw)}
            {explanationCard('The space saving compounds', 'This is exactly the saving you get from arrays and the saving you get from structures, multiplied together. Ten devices × eleven parameters — 110 individually-named flat variables in the worst case — collapse into one declaration line and one structure definition. And because the device is now just an integer index, the same IF/ELSIF block can be wrapped in FOR i := 1 TO 10 DO … Pumps[i].Manuel_Start … Pumps[i].Coil … END_FOR — one block of logic running the entire fleet, however large it grows.', c.green)}
          </div>
        )}

      </div>
    </div>
  );
};
